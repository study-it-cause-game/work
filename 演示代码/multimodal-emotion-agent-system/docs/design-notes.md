# 设计说明

## 系统定位

本项目定位为“面向多模态情感识别的多智能体协同自助系统”课程原型。当前工程使用 React + Vite 实现，重点展示完整业务流程，而不是追求真实模型精度。

## 多智能体流程

1. 用户输入文本或上传多媒体文件。
2. 任务规划智能体识别素材类型并制定执行计划。
3. 数据预处理智能体完成文本清洗、文件类型识别和片段生成。
4. 文本、语音、视觉智能体分别输出单模态情绪结果。
5. 融合决策智能体根据模态质量和置信度生成综合情绪。
6. 解释生成智能体输出可读报告。
7. 自助问答智能体基于报告回答用户问题。

## 情绪类别

当前原型包含五类情绪：

- 积极/开心
- 悲伤/低落
- 愤怒/不满
- 焦虑/紧张
- 中性/平稳

## 可替换模型接口

后续可以将 `src/analysisEngine.js` 中的以下函数替换为真实后端模型接口：

- `analyzeText(text)`
- `analyzeSpeech(files)`
- `analyzeVision(files)`
- `fuseResults(results, plan)`

例如，前端可以通过 `fetch('/api/analyze/text')` 调用后端文本模型，通过 `fetch('/api/tasks/{id}/run')` 启动异步任务。

## 注意事项

当前系统不能用于医学诊断或强制性人员评价。情绪识别结果只适合作为辅助分析参考。
